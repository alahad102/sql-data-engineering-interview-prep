# SQL Diagnostic Test

This folder contains my initial SQL diagnostic test for Data Engineering interview preparation.

The purpose of this test is to evaluate my current SQL level before starting the structured module-based practice plan.

## What This Test Covers

The diagnostic test includes questions from the following SQL areas:

- Basic SELECT queries
- WHERE filtering
- NULL handling
- Aggregation with COUNT and SUM
- GROUP BY and HAVING
- INNER JOIN
- LEFT JOIN
- Anti-join logic
- Subqueries
- Date grouping
- CASE WHEN
- Duplicate detection
- Data quality validation

## Goal

The goal of this diagnostic test is not only to get correct answers, but also to identify weak areas that need more practice.

Based on this test, I found that I need to improve my fluency in:

- SQL execution order
- GROUP BY logic
- WHERE vs HAVING
- COUNT vs SUM
- Date-based aggregation
- CASE WHEN boundary conditions
- Writing queries with less trial and error

## Learning Method

I first attempted the questions on my own using MySQL. After solving, I reviewed the mistakes and updated the corrected solutions.

This diagnostic test will help guide the next modules in this repository.
